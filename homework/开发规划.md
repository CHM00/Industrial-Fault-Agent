# 人工智能体构建第一次作业 — 开发规划

## 一、作业需求梳理

### 核心任务

基于 LangGraph 框架，构建一个计算质数之和的 Agent，完成以下四步流水线：

1. **生成列表**：随机生成5维列表，元素为 1000~10000 之间的正整数，从小到大排列
2. **设计两种方法**：给定正整数 n，计算小于 n 的所有质数之和
3. **计时并放大**：对列表中每个整数，分别用两种方法计算质数之和，通过自定义 Python 装饰器统计耗时，放大10000倍后形成两个新列表
4. **列表相减**：两个时间列表相减并输出

### 示例

```
生成的列表: [1021, 3268, 3468, 4673, 7651]
方法A耗时×10000: [0.10, 0.12, 0.13, 0.14, 0.15]
方法B耗时×10000: [0.11, 0.13, 0.14, 0.16, 0.18]
相减结果: [-0.01, -0.01, -0.01, -0.02, -0.03]
```

## 二、技术方案设计

### 2.1 两种质数求和算法

| 方法 | 算法 | 思路 | 时间复杂度 | 预期速度 |
|------|------|------|-----------|---------|
| 方法A | 逐一判断法（Trial Division） | 对每个数逐一判断是否为质数，是则累加 | O(n√n) | 较慢 |
| 方法B | 埃拉托斯特尼筛法（Sieve of Eratosthenes） | 一次性筛出所有质数，再求和 | O(n log log n) | 较快 |

> 方法A慢、方法B快，这样时间差才明显，相减结果才有意义。

### 2.2 自定义装饰器

```python
import time
import functools

def timing_decorator(func):
    """统计函数执行时间，放大10000倍返回 (结果, 放大后耗时)"""
    @functools.wraps(func)
    def wrapper(n):
        start = time.perf_counter()
        result = func(n)
        elapsed = (time.perf_counter() - start) * 10000
        return result, round(elapsed, 6)
    return wrapper
```

### 2.3 状态定义（AgentState）

```python
from typing import TypedDict, Annotated

class AgentState(TypedDict):
    numbers: list[int]          # 生成的5维正整数列表
    results_a: list[float]      # 方法A的耗时列表（已放大10000倍）
    results_b: list[float]      # 方法B的耗时列表（已放大10000倍）
    difference: list[float]     # 两个列表相减的结果
```

> 不使用 `messages` 字段，因为本作业不涉及 LLM 对话，是纯计算流水线。

### 2.4 节点设计

| 节点 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `generate_numbers` | 随机生成5个1000~10000的正整数，排序 | 无 | `numbers` |
| `compute_with_method_a` | 用逐一判断法计算每个数的质数之和，装饰器计时 | `numbers` | `results_a` |
| `compute_with_method_b` | 用筛法计算每个数的质数之和，装饰器计时 | `numbers` | `results_b` |
| `calculate_difference` | 两个时间列表相减 | `results_a`, `results_b` | `difference` |

> 4个节点 > 2个节点，满足"节点数量≥2"的要求。

### 2.5 图结构

```
START → generate_numbers → compute_with_method_a → compute_with_method_b → calculate_difference → END
```

这是线性流水线，无循环、无条件分支，逻辑清晰。

也可以加入并行计算（方法A和方法B并行执行），但线性流程更简单直观，符合作业要求：

```
START → generate_numbers → compute_with_method_a → compute_with_method_b → calculate_difference → END
```

**并行版本（进阶）**：

```
START → generate_numbers → ┬→ compute_with_method_a ──┬→ calculate_difference → END
                           └→ compute_with_method_b ──┘
```

> 并行版本需要用 `Send` API 或将两个方法的计算结果合并，复杂度更高。建议先用线性版本完成作业。

## 三、核心代码实现

### 3.1 质数求和算法

```python
@timing_decorator
def method_a_sum_primes(n: int) -> int:
    """方法A：逐一判断法求小于n的所有质数之和"""
    def is_prime(num):
        if num < 2:
            return False
        for i in range(2, int(num**0.5) + 1):
            if num % i == 0:
                return False
        return True
    
    return sum(i for i in range(2, n) if is_prime(i))


@timing_decorator
def method_b_sum_primes(n: int) -> int:
    """方法B：埃拉托斯特尼筛法求小于n的所有质数之和"""
    if n < 2:
        return 0
    sieve = [True] * n
    sieve[0] = sieve[1] = False
    for i in range(2, int(n**0.5) + 1):
        if sieve[i]:
            for j in range(i*i, n, i):
                sieve[j] = False
    return sum(i for i in range(2, n) if sieve[i])
```

### 3.2 节点函数

```python
import random

def generate_numbers(state: AgentState) -> AgentState:
    """节点1：随机生成5个1000~10000的正整数，从小到大排列"""
    numbers = sorted(random.randint(1001, 9999) for _ in range(5))
    print(f"生成的数字列表: {numbers}")
    return {"numbers": numbers}


def compute_with_method_a(state: AgentState) -> AgentState:
    """节点2：用方法A（逐一判断法）计算每个数的质数之和并计时"""
    results_a = []
    for num in state["numbers"]:
        result, time_scaled = method_a_sum_primes(num)
        results_a.append(time_scaled)
        print(f"  方法A - n={num}: 质数之和={result}, 耗时×10000={time_scaled:.6f}")
    return {"results_a": results_a}


def compute_with_method_b(state: AgentState) -> AgentState:
    """节点3：用方法B（筛法）计算每个数的质数之和并计时"""
    results_b = []
    for num in state["numbers"]:
        result, time_scaled = method_b_sum_primes(num)
        results_b.append(time_scaled)
        print(f"  方法B - n={num}: 质数之和={result}, 耗时×10000={time_scaled:.6f}")
    return {"results_b": results_b}


def calculate_difference(state: AgentState) -> AgentState:
    """节点4：两个耗时列表相减"""
    diff = [round(a - b, 6) for a, b in zip(state["results_a"], state["results_b"])]
    print(f"\n方法A耗时列表: {state['results_a']}")
    print(f"方法B耗时列表: {state['results_b']}")
    print(f"相减结果: {diff}")
    return {"difference": diff}
```

### 3.3 图构建与运行

```python
from langgraph.graph import StateGraph, END

graph = StateGraph(AgentState)

graph.add_node("generate_numbers", generate_numbers)
graph.add_node("compute_with_method_a", compute_with_method_a)
graph.add_node("compute_with_method_b", compute_with_method_b)
graph.add_node("calculate_difference", calculate_difference)

graph.set_entry_point("generate_numbers")
graph.add_edge("generate_numbers", "compute_with_method_a")
graph.add_edge("compute_with_method_a", "compute_with_method_b")
graph.add_edge("compute_with_method_b", "calculate_difference")
graph.add_edge("calculate_difference", END)

app = graph.compile()

result = app.invoke({"numbers": [], "results_a": [], "results_b": [], "difference": []})
```

## 六、关键设计决策

### 6.1 为什么不用 LLM？

作业要求的核心是"计算质数之和 + 装饰器计时"，属于纯计算任务，不涉及自然语言处理或对话交互。LangGraph 只是作为**流程编排框架**使用，将四个步骤串成一条流水线。这与 LangGraph 官方教程中"不依赖 LLM 的纯计算图"用法一致。

### 6.2 为什么选择这两种算法？

- **逐一判断法**：对每个数做质数判断，O(n√n)，速度较慢
- **埃拉托斯特尼筛法**：一次性筛出所有质数，O(n log log n)，速度较快

两种算法的效率差异明显，放大10000倍后时间差更清晰，相减结果有意义。

### 6.3 为什么用装饰器计时？

作业明确要求"通过自定义的Python装饰器函数统计计算所花时间"，这是关键评分点。装饰器的优势：
- 不侵入业务逻辑
- 统一计时方式
- 可复用于任何函数

### 6.4 节点数量的考虑

4个业务节点（generate_numbers, compute_with_method_a, compute_with_method_b, calculate_difference），远超"≥2"的要求。如果需要减少，最简方案为3个节点：
- `generate_and_compute`（生成列表 + 两种方法计算）
- `compute_both`（分别用两种方法计算并计时）
- `subtract`（相减输出）

但4节点的划分粒度更合理，职责更单一。
