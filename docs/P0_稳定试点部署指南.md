# P0 稳定试点部署指南

## 1. 本阶段交付能力

本阶段把原有单进程 MVP 补强为适合单企业、单实例受控试点的版本：

- LangGraph checkpoint 使用官方 SQLite 后端，专家等待状态可跨服务重启恢复；
- 诊断任务、状态、拥有者、租户、事件和输出路径持久化；
- 支持任务列表、任务详情、取消和失败/中断任务重试；
- 配置化账号登录、HttpOnly 会话 Cookie 和 `viewer/operator/expert/admin` 四级 RBAC；
- 用户与租户级历史记录、报告和流程图访问隔离；
- 每用户滑动窗口限流、全局诊断并发上限和 30 秒排队上限；
- 确定性工业安全预检，高风险任务在检索和 LLM 推理前强制暂停；
- 高风险安全门只能由 `expert` 或 `admin` 明确批准/拒绝，自动模式不能绕过；
- Mermaid 官方解析器改为常驻进程，区分冷启动和热校验超时；
- 存活、就绪和 Prometheus 文本指标接口；
- 固定 Python/npm 依赖、CI、容器 smoke test 和最小安全黄金集。

该版本仍是“受控试点”，不是工业安全认证系统。现场执行必须遵循企业 SOP、法规和授权流程。

## 2. 必填配置

复制 `.env.example` 为 `.env`，至少修改：

```env
LLM_API_KEY=真实模型密钥
LLM_BASE_URL=https://your-compatible-endpoint/v1
LLM_MODEL=your-model

APP_ENV=production
AUTH_ENABLED=true
AUTH_SESSION_SECRET=请替换为至少32字符的随机值
AUTH_USERS_JSON={"admin":{"display_name":"系统管理员","password":"请替换强密码","subject":"admin-1","role":"admin","tenant_id":"factory-a"}}
CHECKPOINT_BACKEND=sqlite
```

账号统一由 `AUTH_USERS_JSON` 配置，也可以把相同 JSON 放入受保护文件并通过 `AUTH_USERS_FILE` 指定。字段含义如下：

- JSON 的键是登录用户名；
- `display_name` 是页面显示名称；
- `subject` 是审计记录中的稳定用户 ID；
- `role` 决定 RBAC 权限；
- `tenant_id` 决定数据隔离范围；
- `password` 便于初次配置，生产环境推荐改用 `password_hash`。

执行 `python security.py`，按提示输入密码即可生成 PBKDF2 密码哈希。`AUTH_SESSION_SECRET`、账号配置和密码哈希只放在环境变量、受保护配置文件或密钥管理系统中，不要提交到 Git、镜像或诊断报告。

`PILOT_API_KEY` 与 `API_KEYS_JSON` 仅作为 CMMS、EAM 等机器系统调用 API 的可选兼容机制，Web 页面不再接收或保存 API Key。

## 3. 角色权限

| 角色 | 能力 |
|---|---|
| viewer | 查看本人/本租户允许的任务、历史和产物 |
| operator | viewer + 发起、恢复普通专家问题、取消、重试任务 |
| expert | operator + 批准或拒绝高风险工业安全门 |
| admin | 查看本租户全部任务和产物、访问 `/metrics` |

机器系统如需兼容 API Key，可使用任一请求头：

```text
X-API-Key: your-key
Authorization: Bearer your-key
```

Web 用户通过用户名和密码登录。服务端签发带 `HttpOnly`、`SameSite=Lax` 属性的会话 Cookie，JavaScript 无法读取会话凭据；左侧显示当前用户、角色和租户。

## 4. 启动与检查

```bash
docker compose up --build -d
```

默认访问地址：

```text
http://localhost:65530
```

健康接口：

- `GET /health/live`：只检查 Web 进程存活；
- `GET /health/ready`：检查任务库、checkpoint、Mermaid validator 和认证配置；
- `GET /metrics`：需要 admin Key，返回 Prometheus 文本指标。

首次 readiness 会启动 Node/Mermaid 常驻校验器，允许最多 25 秒冷启动。后续单次校验默认 8 秒。

## 5. 持久化和备份

Docker 将 `/app/output` 映射到宿主机 `./output`。关键文件为：

```text
output/runtime/checkpoints.sqlite3  # LangGraph 状态
output/runtime/pilot.sqlite3        # 任务、拥有者、状态和审计事件
output/diagrams/                    # Mermaid 与报告
output/memory/                      # 诊断记忆
```

备份时应同时备份整个 `output`，避免任务表、checkpoint 和产物版本不一致。SQLite 适合当前单实例试点，不支持多个应用副本共享写入；需要横向扩展时应迁移到 PostgreSQL/Redis 官方 checkpoint 和集中式任务库。

## 6. 任务状态

主要状态如下：

```text
queued -> running -> waiting_safety -> running
                  -> waiting_feedback -> running
                  -> completed / denied / failed / cancelled
服务重启时 running -> interrupted -> retry 创建新任务
超过 SESSION_TTL_SECONDS -> expired
```

API：

- `GET /api/jobs`：任务列表；
- `GET /api/jobs/{session_id}`：详情和审计事件；
- `POST /api/jobs/{session_id}/cancel`：取消；
- `POST /api/jobs/{session_id}/retry`：重试终态/中断任务；
- `POST /api/resume`：恢复专家问题或安全审批。

等待专家反馈的任务依赖持久化 checkpoint，可在服务重启后使用原 `session_id` 恢复。服务崩溃时正在执行的外部请求不能安全续跑，会标记为 `interrupted`，应通过 retry 创建新任务。

取消采用协作式取消：系统会在下一个 LangGraph 节点事件处停止交付和保存产物，但已经发出的第三方 LLM/搜索 HTTP 请求可能继续到自身超时。

## 7. 高风险安全审批

规则当前覆盖：

- 高压、带电、电弧和母线；
- 可燃、爆炸、燃气和泄漏；
- 高温、蒸汽和压力能量；
- 旋转机械；
- 旁路保护、屏蔽联锁和强制输出；
- 受限空间、有毒和缺氧。

风险为 `high` 或 `critical` 时，图在内部检索、外部研究和 LLM 推理之前暂停。恢复请求必须由 expert/admin 提交：

```json
{
  "session_id": "example",
  "feedback": "已完成停机、隔离、验电和 LOTO，批准仅进行离线诊断。",
  "approved": true
}
```

拒绝时传 `approved: false`，任务进入 `denied`，不会生成或保存诊断流程图。

## 8. 质量门

本地开发：

```bash
pip install -r requirements-dev.txt
npm ci
pytest tests -q -p no:cacheprovider --basetemp=output/pytest-local
python evaluation/run_p0_evaluation.py
```

CI 同时执行：

- Python 编译；
- 全部单元/契约/集成测试；
- 安全黄金案例；
- Docker 构建和 `/health/ready` smoke test。

`evaluation/golden_safety_cases.json` 只是 P0 确定性安全门基线。后续仍需加入企业专家标注的真实故障案例，评估根因命中、安全步骤覆盖、SOP 引用和专家修改率。
