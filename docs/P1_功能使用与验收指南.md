# P1 功能使用与验收指南

## 1. 本阶段交付

P1 将 P0 的“可稳定运行诊断”扩展为“可由设备操作员与专家共同使用的业务闭环”，实现以下八项能力：

| 编号 | 能力 | 当前实现 |
|---|---|---|
| P1-1 | SOP 知识库管理 | txt/md/csv/tsv/json/pdf/docx/xlsx/xlsm 上传解析、确定性分块、版本、角色权限、生效/失效、只检索当前有效版本 |
| P1-2 | 设备资产与结构化表单 | 设备编号、类型、厂商、型号、固件、位置、关键度，以及报警码、工况、测点、维修史 |
| P1-3 | 历史案例 RAG | 诊断结束自动生成待确认案例；专家回填根因、处理措施、结果、评分和可信度后参与相似案例召回 |
| P1-4 | 步骤证据映射 | Mermaid 节点映射受控 SOP、历史案例和外部来源，输出位置、引用摘录、置信度与待复核标记 |
| P1-5 | 流程编辑与审批 | Mermaid 源码实时预览、不可变版本、统一 Diff、批准、拒绝、签名发布；任务发布后才进入完成态 |
| P1-6 | 现场文档导出 | Word 报告、PDF 报告、UTF-8 BOM CSV 检查单 |
| P1-7 | 诊断任务中心 | 草稿、排队、运行、待反馈、待安全审批、待流程审批、完成、失败、取消和归档；失败任务可重试 |
| P1-8 | 质量成本看板 | 任务数、审计通过率、耗时、修订、搜索、Token、估算成本，以及按设备和模型聚合 |

## 2. 角色与权限

| 操作 | viewer | operator | expert | admin |
|---|---:|---:|---:|---:|
| 查看有权限的资产、任务和知识 | 是 | 是 | 是 | 是 |
| 创建设备、草稿、诊断和流程新版本 | 否 | 是 | 是 | 是 |
| 上传/更新/停用 SOP | 否 | 否 | 是 | 是 |
| 确认案例、批准/拒绝/发布流程 | 否 | 否 | 是 | 是 |
| 查看质量成本看板 | 否 | 否 | 是 | 是 |

所有业务表均带 `tenant_id`，API 同时检查租户与任务所有者；专家可在本租户内处理待审批任务，不能跨租户读取数据。

## 3. 推荐试点流程

1. 管理员在 `.env` 配置 API Key、数据库和模型服务，启动应用。
2. 专家打开“SOP 知识库”，上传有效 SOP；同一文档更新时填写旧文档 ID，系统生成下一版本并停用旧版本索引。
3. 操作员打开“设备资产”，录入试点设备。
4. 操作员选择设备，填写报警码、工况、测点 JSON 与近期维修史，然后开始诊断。
5. Agent 将受控 SOP 和可信历史案例与原 Milvus 检索结果一起融合，并为流程步骤建立证据映射。
6. 专家在“编辑/审批流程”中校对流程，保存新版本、查看 Diff、批准并发布。
7. 现场人员导出 Word/PDF 报告或检查单执行；专家在“回填处理结果”中确认根因和实际措施。
8. 专家从“质量成本看板”观察试点表现，并在任务中心归档完成任务。

## 4. 配置

```env
# 单文件和提取文本限制
KNOWLEDGE_MAX_UPLOAD_BYTES=15728640
KNOWLEDGE_MAX_EXTRACTED_CHARS=2000000

# 按模型供应商账单填写；单位为每 100 万 Token
LLM_INPUT_COST_PER_1M=0
LLM_OUTPUT_COST_PER_1M=0

# Windows 通常无需设置；Linux Docker 已安装文泉驿字体
# PDF_CJK_FONT=C:/Windows/Fonts/msyh.ttc
```

受控知识、案例和指标使用 `PILOT_DB_PATH` 指向的 SQLite 文件。单实例试点应持续备份 `output/runtime`；不要同时让多个容器写同一个 SQLite 卷。

## 5. 关键 API

| 方法与路径 | 最低角色 | 用途 |
|---|---|---|
| `GET/POST /api/assets` | viewer/operator | 列表/创建设备 |
| `PATCH /api/assets/{id}` | operator | 更新或停用设备 |
| `POST /api/knowledge/documents` | expert | 请求体直接传文件字节；查询参数传标题、文件名、权限、旧文档 ID 和变更说明 |
| `GET /api/knowledge/documents/{id}/versions` | 按文档权限 | 查看版本链 |
| `PATCH /api/knowledge/documents/{id}/status` | expert | active、inactive 或 archived |
| `GET /api/knowledge/search?q=...` | 按文档权限 | 验证有效版本召回 |
| `POST /api/jobs/draft` | operator | 保存草稿 |
| `POST /api/jobs/{id}/start` | operator | 启动草稿 |
| `GET /api/jobs` | viewer | 按状态或关键词查看任务 |
| `POST /api/procedures/{session}/versions` | operator | 校验 Mermaid 后保存流程版本 |
| `GET /api/procedures/{session}/diff` | viewer | 比较两个版本 |
| `POST /api/procedures/{session}/adopt` | operator | 按选中的行级差异项生成组合版本 |
| `POST .../approve|reject|publish` | expert | 审批生命周期 |
| `POST /api/cases/{session}/confirm` | expert | 确认案例结果 |
| `GET /api/jobs/{session}/export` | viewer | `docx`、`pdf` 或 `checklist` |
| `GET /api/dashboard/quality` | expert | 质量与成本指标 |

## 6. 验收清单

- 上传 SOP v1 后可检索；以相同文档 ID 上传不同内容后出现 v2，v1 为 `superseded`，搜索只返回 v2。
- viewer 无法搜索仅允许 operator/expert 的文档；停用文档后所有角色均无法召回。
- 选择设备开始诊断后，结果包含设备上下文、受控知识命中、历史案例命中和步骤证据。
- 诊断生成后任务状态为 `pending_approval`；可在相邻版本 Diff 中逐条勾选采纳并生成新版本；草稿版本不能直接发布，批准后才能发布，发布后任务为 `completed`。
- Word 可由 Office 打开；PDF 中文可显示；检查单包含每个流程节点、风险、证据和现场填写列。
- 回填案例后，以根因或处理措施关键词启动新诊断能够召回该案例，并显示可信度。
- 看板任务数、Token、搜索次数和耗时随新诊断增加；单价为零时成本保持零。
- 不同租户使用相同设备编号不会冲突，且彼此不能读取资产、知识、案例或任务。

## 7. 当前边界

- 受控知识检索当前采用 SQLite 上的确定性文本相似度，适合单实例、中小规模试点；大规模知识库应迁移到具备元数据过滤的集中式向量索引。
- 流程编辑当前是 Mermaid 源码 + 实时可视化，不是拖拽式图编辑器；版本 Diff 为文本级统一 Diff。
- 外部来源的位置通常是 URL，无法保证页码；受控 PDF 的位置可标注页码，Excel 可标注工作表。
- 成本为按 Token 与配置单价计算的估算值，应与供应商账单对账。
- 现场检查单为文件导出，尚未直接对接 CMMS/EAM 工单系统。
